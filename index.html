The box model. What is it?

There are four parameters that each HTML element has: content, padding, border and margin. To understand these attributes we can imagine a simple box contains some content, this content is located at some distance from the walls of the box (this is the padding attribute) and this box itself has red outlines (this is the border attribute). The box itself is in the center of the room at some distance from the other boxes. This given distance is called the margin.
(У каждого элемента HTML есть 3 параметра: контент, падинг, границы и маржин. Чтобы понять эти атрибуты просто представим коробку в комнате, где много таких коробок. Каждая коробка - это HTML тег, коробка содержит какой-то контент. Контент не облегает плотно все стенки коробки, значит, между контентом и стенками коробки(границами) есть какое-то пространство. Оно называется паддинг. Мы можем его менять как душе угодно, меняя паддинг, контент будет находится дальше от стенок(границ) коробки. Коробка может быть обрамлена, например, в красный цвет). Расстояние между другими коробками (HTML элементами) мы тоже можем задать, как и в реально жизни, мы можем передвинуть коробку подальше, или поближе. Это расстояние между другими HTML элементами называется маржин

Margin collaps. What is it?

Let's say there are three other blocks in one div block. There is a 20 pixel padding between them. Let's assume that the second block should have a padding of 60px from the top. We give it this padding separately, and instead of adding these 20 pixels of padding (as we specified for all blocks) with these 60 pixels of padding specifically for the second block, this does not happen. This 60px padding overlaps the 20px padding.
Допустим в одном блоке див три других блока. Между ними отступ в 20 пикселей. Предположим, что у второго блока отступ должен быть 60 пикселей сверху. Мы ставим ему отдельно этот отступ и вместо того, чтобы сложить эти 20 пикселей отступа (как мы указали для всех блоков) с этими 60 пикселями отступа специально для второго блока, этого не происходит. Этот отступ в 60 пикселей перекрывает отступ в 20 пикселей.

Overflow. What is it? How we can fix this?

Sometimes it happens that the text gets out of the block in which it is located. It can be not only text, but also an image, video, etc. We can control these cases by using the overflow selector. There are three parameters: hidden/scroll/visible . In the first case, the element outside the block will be hidden. In the second case, a scroll bar is added so that we can fully view this element. In the third case, the text will be visible despite the fact that it goes beyond the block.

Иногда случается так, что текст вылезает за рамки блока в котором он находится. Это может быть не только текст, но и изображение, видео и т.д. Мы можем контролировать эти случае использовав селектор overflow. Есть три параметра: hidden/scroll/visible . В первом случае элемент за рамками блока будет скрыт. Во втором случае добавляется полоса прокрутки, чтобы мы могли полностью просмотреть этот элемент. В третьем случае текст будет виден несмотря на то, что он выходит за рамки блока.

Selector box-sizing with value border-box. What does it mean?

This is one of the properties that should be set by default, since it does not allow the element to be enlarged more than allowed. For example, using the padding selector, there is a chance to increase the element, since if the content comes too close to the boundaries of the block, the block must expand these boundaries. The box-sizing selector with the border-box value does not allow this. It guarantees the preservation of the specified proportions of width and height. For example, if the width is 500 pixels and we use padding, then in any case the element will remain 500 pixels in size, even with a high padding value.

Это одно из свойств, которые следует ставить по умолчанию, так как оно не позволяет увеличение элемента больше допустимого. Например, использовав селектор padding есть вероятность увеличить элемент, так как если содержимое подойдёт слишком близко к границам блока, блок должен расширить эти границы. Селектор box-sizing со значением border-box этого не позволяет. Он гарантирует сохранение заданных пропорций width и height. Например, если width 500 пикселей и мы используем padding, то в любом случае элемент останется размером в 500 пикселей, даже при высоком значении padding.
